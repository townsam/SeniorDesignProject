Capstone Assessment

My senior design project is a Unity-based puzzle game where a collection of actors behave unpredictably, and the player guides their behavior until every actor completes a shared objective. From my academic perspective, this project sits right at the intersection of reinforcement learning, data analysis, and game development, which are the areas of computer science I enjoy the most. The core idea is that all actors start with randomized movement patterns, and the player chooses which ones to replicate so that the entire group gradually converges toward the correct behavior. The underlying system mixes game logic, actor simulation, and a data-driven learning pipeline, which makes it a perfect challenge for applying both my coding background and analytical mindset. I also like that the project mirrors real machine learning concepts such as imitation learning and agent convergence, but in a visual, interactive setup. Overall, this project feels like a natural fit because it lets me blend creativity with technical depth while building something that’s both fun and conceptually meaningful.

My coursework in computer science is a major driver in how I approach this project because so many of the classes I’ve taken directly map onto the systems we need to build. Courses like CS2028C Data Structures and Algorithms strengthened my understanding of how to efficiently manage actor states, game data, and simulation events behind the scenes. Machine learning topics from CS4033 helped me understand how reinforcement tools can map onto cost functions and behavior shaping. Even electives like human computer interaction shaped how I think about the player’s experience with selecting actors and giving commands. Everything from these classes influences how I design features, debug issues, and evaluate whether the system performs the way we expect.

My co-ops also play a huge role in how I contribute to the project because they taught me skills that go way beyond classroom knowledge. Working as a Digital Manufacturing Intern at Owens Corning gave me hands-on experience with large datasets, anomaly detection pipelines, and building analytics logic that refines raw data into actionable insights. That experience maps directly onto my role designing the game’s data logging system and analytics pipeline that helps track actor performance. At Kroger Technology, my work in portfolio operations sharpened my ability to automate processes, structure information, and manage technical systems at scale, which helps me think about maintainability and scalability in our game’s backend. Both co-ops strengthened my technical communication skills because I constantly had to explain complex ideas to people with different backgrounds. I also learned the importance of testing, iteration, and quality control, which directly affects how I plan to validate the reliability of our game systems. These real-world experiences help me stay organized and intentional during development, especially when working across different modules like ML integration, Unity behavior systems, and data structures.
I’m genuinely excited to work on this project because it combines everything I enjoy about CS from problem solving, machine learning, game design, to data analytics. The idea of taking something chaotic like randomized actors and slowly shaping them into a coordinated solution feels satisfying from both a technical and creative standpoint. My preliminary approach starts with building a strong data foundation so every actor’s behavior, performance, and outcomes can be analyzed as part of the learning loop. From there, our team plans to iterate through tool prototypes, ML integration, and level mechanics, always testing how each decision affects the puzzle design. I want the final game to feel intuitive for players while still teaching real ML concepts through gameplay. My motivation comes from wanting to build something unique, something that blends entertainment with algorithmic thinking in a way that feels approachable and fun.

By the end of the project, I expect us to have a fully functional game with multiple puzzle levels, a complete set of reinforcement tools, and a working pipeline where ML influences actor behavior and vice versa. My personal accomplishments will focus on delivering a clean, reliable data storage and analytics system, as well as ensuring the integration between game events and ML outputs is smooth. I will evaluate my contributions based on clarity, consistency, and how well each system supports player interaction without breaking immersion. I’ll know I’ve done a good job if the data tools feel invisible to the player but essential to the game’s internal logic, and if the ML integration actually creates meaningful variations in actor behavior. Another marker of success is whether the team can rely on my systems without worrying about failures or bottlenecks. Ultimately, I want to finish the project feeling confident that I contributed core components that the rest of the game depends on.
